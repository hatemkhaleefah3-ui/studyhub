Part 4 — Feature: Exam Schedule
4.1 — Exam Schedule fields
Name
Start date / End date — overall range
Exams (add one or more), each with:
Link to an existing subject from the Subjects page → the subject’s name becomes the exam’s name (no separate name field)
Exam date
Exam time — defaults to 9:00 AM if left unset

Part 5 — Feature: Review Schedule
5.1 — Review Schedule fields
Name
Start date / End date — overall review period
Subjects (add one or more), each with:
Link to an existing subject from the Subjects page → its name is reused (no separate name field)
Its own start date / end date, which must fall within the overall review schedule’s range
5.2 — Calendar coloring: review schedule days
Every day within a review schedule’s range → transparent yellow
If a day is both a review day and an exam day → stays transparent red (exam color wins over yellow)
5.3 — “Next Review” section (dashboard) — rotating active subject Positioned above the exam schedule section:
The subject whose start date is closest to the review schedule’s start date is shown first
While active, that subject’s date range is highlighted transparent dark green on the calendar (overrides yellow; exam red still wins if overlapping)
Once the timeline passes that subject’s end date, the section automatically switches to whichever remaining subject has the next-closest start date; its range becomes the new dark-green highlight
This rotation continues, subject by subject, until the review schedule’s end date is reached

Part 6 — Calendar: Single “Most Important” Schedule Highlight
6.1 — Calendar: single “most important” schedule highlight Among all active schedules (exam and/or review), determine the most important one via the importance setting. Only that schedule’s days are highlighted on the calendar with its assigned transparent color — other, less-important schedules do not simultaneously display their highlight on the same calendar.
6.2 — Updated calendar color priority (highest to lowest)
Individual exam day → red (always takes priority, regardless of importance ranking)
The single most-important schedule’s days → its assigned transparent color (per 6.1)
Active “Next Review” subject day → dark green (if not already overridden by a higher rule above)
General review schedule day → yellow
No event → no highlight
